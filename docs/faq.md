# TSAM FAQ

This FAQ is intended to reduce ambiguity around the Trinity Space Arbitration Model (TSAM) repository and its current research status.

## 1. What is TSAM?

TSAM is a reality-constrained cosmological research framework. It is used to organize, document, and test observational anomalies across multiple astrophysical datasets under explicit validation rules.

The framework does not claim that a proposed structure is established simply because it is internally coherent. Observational constraint, reproducibility, and cross-dataset consistency remain the primary standards.

## 2. Is TSAM a completed cosmological model?

No. TSAM should be read as an evolving research framework, not as a closed or finalized replacement for existing cosmological models.

Some components are theoretical hypotheses. Some are data-audit procedures. Some are public archive records. These categories should not be mixed.

## 3. What is the main methodological difference?

TSAM gives priority to external observational constraint over internally self-consistent model growth.

The core rule is simple: a proposed interpretation must remain provisional unless it survives fixed-rule checks, independent data comparison, and falsification attempts.

## 4. What datasets are currently relevant?

The current public roadmap focuses on:

- CMB directional and large-scale anisotropy features
- QSO redshift-space structure
- LRG and galaxy-density fields
- SPARC galaxy rotation residuals
- Solar-system scale boundary and local-frame consistency tests
- JWST high-redshift anomaly discussions

## 5. What is the role of Zenodo DOI records?

Zenodo DOI records preserve public research snapshots. They should be treated as versioned archive records, not as automatic validation of every theoretical claim.

The DOI list in the repository README is maintained as a transparent academic archive chain.

## 6. What does "reality-constrained" mean here?

It means that observational data has priority over conceptual elegance, mathematical closure, or self-consistent narrative expansion.

A TSAM-related claim should be downgraded, revised, or frozen if it cannot be tested against external evidence.

## 7. Can other researchers reproduce the work?

That is the long-term goal. Current documentation is being organized to separate:

- theoretical descriptions,
- validation principles,
- data-audit tracks,
- reproducibility requirements,
- and future tooling such as WangTools.

## 8. How should external readers engage?

The most useful engagement is not agreement or rejection at the slogan level. The preferred engagement mode is:

1. Identify a specific claim.
2. Check whether it is labeled as hypothesis, audit result, or archive note.
3. Ask what dataset and rule set support it.
4. Propose a falsification or replication test.

## 9. What is not claimed?

This repository does not claim that every TSAM hypothesis has already been fully validated. It also does not claim that existing cosmological models are replaced by declaration.

The repository records a research program and its evolving validation structure.

## 10. What is the next priority?

The immediate priority is to strengthen the public repository structure, document clear claim boundaries, and prepare reproducible validation tracks before broader community expansion.
