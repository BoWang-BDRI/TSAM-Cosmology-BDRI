# TSAM Validation Principles

## 1. Reality-Constrained Reasoning

TSAM uses reality-constrained reasoning as its primary methodological rule. A theory may be internally elegant, but it must remain subordinate to observational data.

This means that TSAM-related claims should be evaluated by external data constraints rather than by internal narrative coherence alone.

## 2. Provisional Assumption Labeling

Unverified assumptions must be marked as provisional.

A provisional assumption may be useful for building a test, but it should not be converted into a hard definition before direct or indirect observational support is available.

Examples of provisional categories:

- Proposed boundary structures.
- Directional axes.
- Cross-epoch coherence claims.
- Redshift reinterpretation mechanisms.
- Global topology assumptions.

## 3. Freeze-Review Cycle

Model growth must be controlled by freeze-review cycles.

A freeze-review cycle has four steps:

1. Freeze the current claim or model state.
2. Define the external validation target.
3. Test against observational data.
4. Either retain, revise, downgrade, or reject the claim.

This prevents uncontrolled expansion of self-consistent but non-falsifiable theory.

## 4. Cross-Dataset Priority

Single-dataset pattern detection is not enough.

A structure becomes more important only if it survives independent checks across different data classes, such as:

- CMB maps.
- Quasar catalogs.
- LRG / galaxy-density surveys.
- SPARC galaxy rotation data.
- Solar-system or local dynamical constraints.

## 5. Residual-First Analysis

Residual fields are treated as diagnostic evidence.

The key question is not whether a preferred model can fit a dataset after sufficient parameter adjustment. The key question is whether residual structures reveal systematic, repeatable, and externally constrained behavior.

## 6. Downgrade Rule

If a TSAM-related claim fails a defined validation test, the claim should be downgraded.

Possible downgrade states:

- Confirmed working hypothesis.
- Provisional hypothesis.
- Pattern candidate.
- Dataset artifact candidate.
- Rejected or retired claim.

## 7. Anti-Hallucination Boundary

The project explicitly rejects unconstrained speculative growth.

A claim should not be strengthened merely because it fits a preferred story. It should only be strengthened when external evidence supports it.

## 8. Public Documentation Standard

Public repository updates should distinguish between:

- Archived papers.
- Active hypotheses.
- Validation methods.
- Data-audit results.
- Speculative future directions.

This keeps the project readable, falsifiable, and professionally auditable.
